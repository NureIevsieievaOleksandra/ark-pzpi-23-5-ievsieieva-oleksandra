3 ОПИС ВИКОНАНОЇ РОБОТИ

	Правила оформлення програмного коду на мові Object Pascal.
Ці правила важливі для того, щоб програма була читабельною, зрозумілою та зручною в підтримці — не лише для автора, а й для будь-якого іншого розробника.
Object Pascal — це сучасна версія Pascal, створена компанією Borland, а зараз її розвиває Embarcadero Technologies.
Перше правило — акуратність у відступах і вирівнюванні.
У Pascal дуже важливо, щоб структура коду була зрозумілою візуально.
Використовуйте два або чотири пробіли для відступів — не табуляцію.
Кожен блок коду всередині begin і end має бути вирівняний.
Як бачимо на прикладі [Додаток В]:
зліва — код без відступів, читати його складно;
справа — той самий код, але оформлений охайно, і його структура відразу зрозуміла.
Наступне правило — іменування.
У Object Pascal прийнято використовувати стиль CamelCase для назв змінних і процедур, тобто кожне нове слово починається з великої літери: наприклад, StudentCount або CalculateAverage.
Константи, навпаки, записуються великими літерами — наприклад, MAX_STUDENTS.
Уникайте коротких або незрозумілих назв, як a або tmp.
Хороше ім’я має підказувати, що саме зберігає змінна.
Коментарі — це пояснення в коді, які не виконуються, але допомагають зрозуміти логіку.
В Object Pascal можна писати коментарі кількома способами: однорядкові — через //, багаторядкові — у фігурних дужках { } або в (* ... *).
Але коментарі мають бути змістовними.
Не потрібно писати очевидне, наприклад: // додаємо 1.
Краще коротко пояснити мету: “Лічильник наступного елемента”.
У Pascal код організовується в модулі (unit).
Кожен модуль має чітку структуру: у розділі interface розміщуються оголошення функцій і процедур, у розділі implementation — їхня реалізація.
На початку модуля бажано додати коментар із коротким описом його призначення.
Усі функції та процедури потрібно оформлювати однаково.
Імена починаються з великої літери, а перед кожною функцією бажано писати короткий коментар, що вона робить.
Наприклад:
“Функція обчислює суму двох чисел”.
Такий підпис зекономить час тим, хто читатиме ваш код пізніше.ак легше зрозуміти, для чого створений цей код, навіть без детального аналізу.
Підсумуємо основні правила оформлення коду в Object Pascal: Використовуйте 2–4 пробіли для відступів; Вирівнюйте пари begin і end; Імена змінних — у CamelCase, константи — великими літерами; Додавайте пробіли навколо операторів; Коментуйте логіку, а не очевидні дії; Кожен модуль і функцію супроводжуйте коротким описом; Дотримуйтеся єдиного стилю по всьому проєкту.
Дотримання цих правил робить ваш код не лише правильним технічно, а й красивим, зрозумілим і професійним.
